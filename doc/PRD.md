# 需求文档 – RAG + 生成式人工智能服务

### 统一业务场景与全球约束

在内部知识库（员工手册、合规指南、技术规格、架构文档）之上构建一个检索增强型问答系统。该语料库为双语（中文/英文），并包含一小部分扫描版PDF文件。该系统须支持多轮对话，并能提供有据可查、引用支撑的答案。

候选人可选择任何技术栈。关键的技术选择须在交付成果中以清晰、定量的证据加以论证。

### 全局约束（适用，除非以下另有说明）

- **API与响应**：接受自然语言问题，并返回基于检索内容的答案。
- **延迟**：90%的用户问题必须端到端在10秒内返回。
- **成本**：提供每1,000次调用的令牌成本估算，以及敏感性分析。顶级\_k, 重新排序器 开/关，以及温度（≥ 3 档）。
- 质量指标：RAG：忠实性 ≥ 0.85（请定义您的评分标准），上下文精确度 ≥ 0.70。
- **日志与追踪**：设计一套适用于生成监控和问题诊断的日志系统。
- **安全性**：最低限度的提示注入防御；基本的个人身份信息处理；答案必须严格依赖于检索到的上下文。

### 目标

交付一个基于RAG的最小可行质量保证系统，具备基本的多轮对话上下文、引用支持的答案，以及可实现问题诊断与迭代的基础日志记录功能。

### 功能需求

1. RAG问答
   - 对内部文档进行质量保证；答案须基于所检索的资料，并附上引用。
2. 多轮对话
   - 支持在同一会话中保持对话连续性，无需用户重复之前的内容。

### 非功能与定量指标

1. 正确性与质量
   - 在您自己的评估集上，整体答案准确率≥80%；同时满足上述忠实性和上下文精确度阈值。
2. 可进化性
   - 面向迭代改进的设计（扩展文档、检索/生成策略微调、指标/日志增强）。
3. 日志记录与可观测性
   - 日志必须足以进行性能分析和生成质量故障排除（包括示例日志）。
4. 鲁棒性
   - 当检索未返回任何结果或相似度极低时，应返回一条解释性的拒绝答复，并清晰地说明当前的能力边界。

### 可交付成果

- 可运行的演示（本地或云端）。
- 完整源代码和自述文件（运行/配置）。
- 200–500 单词 设计说明，解释关键选择与权衡取舍。
- 包含指标表格和已脱敏个人身份信息（PII）样本日志的评估摘要。
